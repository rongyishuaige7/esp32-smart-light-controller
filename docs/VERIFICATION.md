# 验证说明

> 状态日期：2026-07-18

## 一键本地门禁

```bash
bash scripts/verify.sh
```

脚本会把当前候选导出到临时目录，再执行：

1. 敏感信息、私钥、凭据、私网地址、本机路径、构建产物、压缩包和大文件检查；
2. 必需公开文件、固定依赖、BOM、接线 SVG、状态边界与公开范围检查；
3. 不依赖硬件的源码契约测试；
4. PlatformIO ESP32 干净构建；
5. 构建后的二次敏感信息和结构检查。

它不会修改桌面原工程、烧录 ESP32、连接真实 Wi‑Fi、写入真实凭据、清除 NVS、控制真实 LED、读取真实 BH1750 或证明网络/硬件行为。

## 构建证据规则

原始工程曾存在 `.pio/` 构建缓存，但该缓存及其时间戳不作为公开候选证据。本轮必须先在当前候选运行 `scripts/verify.sh`，然后在最终 `main` commit 上由 GitHub Actions 重新构建。只有 Actions 成功且 run 的 commit 与公开默认分支 HEAD 完全一致，才可把它写为构建证据。

构建成功仍不等于烧录、上电、本机凭据连接、HTTP、NVS、LED、BH1750、倒计时、自动模式、网络安全或长期稳定性验证。

## 当前真机复测清单

更新“当前真机已复测”前，必须记录测试日期、完整 Git commit、精确 ESP32 板型/Flash/USB、每个模块型号、供电、极性、接线、电平、每项通过/失败/未测和脱敏证据。至少验证：

- [ ] 当前 commit 能烧录、启动、复位，且四路 GPIO / 实际 LED 在上电和复位时不会出现未预期输出；
- [ ] 四个 LED 的极性、限流或恒流驱动、每路实际电流、外部低压电源、公共地和长时间温升；
- [ ] BH1750 的型号、供电、I²C 地址、上拉、正常读取、拔线/失败路径和 API `unavailable` 语义；
- [ ] 默认公开配置不创建接入点、不尝试连接 Wi‑Fi 且不启动 HTTP；本机凭据连接、断电/重连与凭据不被串口/页面/媒体回显；
- [ ] 在隔离可信局域网测试 `GET /api/status`、合法/超限/分片 JSON 的两条 POST、字段错误和 404；
- [ ] 分别验证四路手动开关、五种倒计时选项、倒计时到零、自动模式四路联动与自动模式覆盖倒计时；
- [ ] 传感器不可用时启用/关闭自动模式，确认没有基于无效读数产生物理输出变化；
- [ ] 30–60 分钟受控运行，观察 Wi‑Fi、NVS、内存、Flash、I²C、GPIO、LED 驱动温升和异常重启；
- [ ] 照片、视频、屏幕录制和日志去除 SSID、密码、私网 IP、MAC、二维码、账号、绝对路径、位置、EXIF/GPS 和其他个人信息。

真机复测只能证明记录中可见的当次硬件行为，不能把本项目升级成市电、公共照明、应急照明、安防、节能认证、可靠远程控制或无人值守产品。

# 硬件与接线边界

本文只根据当前公开固件的引脚常量整理接口，**不是**原理图、PCB、实物照片、已验证接线或电气认证。原始工程当前未提供可公开的实物照片、视频、原理图、PCB、Gerber、制造文件或与公开 commit 绑定的真机记录；因此任何模块型号、极性、供电和实际接法都必须在通电前自行核对。

## 源码推导接线边界

`hardware/wiring-diagram.svg` 仅用于解释上述源码接口；它不是 PCB 原理图，也不能证明实物按照该图接线。

1. **断电接线。** 调整导线、模块、电源、LED 或传感器前，先断开 USB 与外部供电。
2. **只接低压负载。** 本固件只适合低压 LED 教学演示。不得把 GPIO、LED 模块、继电器、驱动板或本项目接入市电、公共照明、应急照明、安防、消防、生产控制或其他高后果设备。
3. **GPIO 不能替代功率驱动。** ESP32 GPIO 是 3.3 V 逻辑输出，不是电源或恒流驱动。多个 LED、较大电流、长线或外部电源场景需要按实物设计合适的限流、晶体管/MOSFET 驱动、保护与独立电源；本仓不提供该设计。
4. **确认极性与上电行为。** 当前公开固件在 `setup()` 中先将四路输出写为 `LOW`，随后才读取保存的 NVS 状态。真实板级上电、复位或掉电瞬间的引脚电平、驱动极性和 LED 状态仍取决于具体电路，必须实测。
5. **确认供电和共地。** ESP32、BH1750、LED 驱动与外部低压电源需要以模块规格为准，并在需要时建立可靠公共地。不要把超过 ESP32 输入/输出容限的电压直接接入 GPIO。
6. **传感器是可选项。** 没有 BH1750，手动四路控制仍是源码功能；自动模式不会基于不存在或失败的传感器读数改变 LED 输出。

## 当前缺失或待确认的实物事实

- ESP32 板型、Flash、USB 芯片、稳定供电、实际烧录方式与当前 commit；
- 四路 LED 的精确器件、极性、限流值、驱动拓扑、额定电流与低压电源；
- BH1750 的模块、电压、I²C 地址、上拉与实际读数；
- 本机凭据连接、局域网访问、默认无凭据不启动 HTTP、倒计时、自动模式与传感器失效路径；
- 实物照片、演示视频、原理图、PCB、Gerber、制造文件与日期化真机复测记录。

因此当前只能表述为：**源码来源已确认，公开候选的无硬件门禁与固件构建可验证；当前真机复测尚未执行。**
